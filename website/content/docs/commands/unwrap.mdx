# `unwrap`

Unwrap data concealed by a wrapping token.

<CodeBlockConfig hideClipboard>

```shell-session
$ vault unwrap <flags> <wrap_token>

$ vault unwrap [-help | -h]
```

</CodeBlockConfig>

## Description

`vault unwrap` fetches and unwraps data in Vault previously wrapped with the
provided token. You can interact with `vault unwrap` results the same way you
interact with results from running `vault read` on the same, non-wrapped data.

Before returning the unwrapped results, Vault validates the provided token,
confirms that the token has sufficient permission to access the requested data,
and logs the request/response.

<Tip title="Related API endpoints">

  WrappingUnwrap - [`POST:/sys/wrapping/unwrap`](/vault/api-docs/system/wrapping-unwrap)

</Tip>

## Examples

Unwrap data in the cubbyhole plugin using an explicit wrap token:

```shell-session
$ vault unwrap 3de9ece1-b347-e143-29b0-dc2dc31caafd
```

Unwrap data in the cubbyhole plugin using the current client token:

```shell-session
$ vault login 848f9ccf-7176-098c-5e2b-75a0689d41cd
$ vault unwrap
```
