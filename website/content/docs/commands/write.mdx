# `write`

Write (`POST` or `PUT`) data to an arbitrary path.

<CodeBlockConfig hideClipboard>

```shell-session
$ vault write [flags] <write_path> <write_data>

$ vault write [-help | -h]
```

</CodeBlockConfig>

## Description

`vault write` takes arbitrary data as key/value pairs, wraps the associated HTTP
`POST` or `PUT` command, and writes the data to an arbitrary mount path, Vault
API endpoint, or plugin API endpoint.

The `write` command accepts [key/value pairs](/vault/docs/commands#passing-data)
as inline parameters or from the `data` parameter of a JSON blob in a file or
`stdin`.

## Examples

Write to the `userpass` plugin API endpoint,
[`/auth/{plugin_mount_path}/users/{name}`](/vault/api-docs/auth/userpass#create-update-user)
with inline key/value pairs:

```shell-session
$ vault write                   \
    /auth/userpass/users/alice  \
    token_ttl="8h"              \
    token_max_ttl="12h"
```


### Write from a file

Write to the `aws` plugin API endpoint,
[`/{plugin_mount_path}/roles/{name}`](/vault/api-docs/secret/aws#create-update-role)
with data from a file on disk:

```shell-session
$ vault write aws/roles/ops policy=@policy.json
```

### Write from `stdin`

Write to the `consul` plugin API endpoint,
[`/{plugin_mount_path}/config/access`](/vault/api-docs/secret/consul#configure-access),
to configure Consul access with data read from `stdin`:

```shell-session
$ echo $MY_CONSUL_TOKEN | vault write consul/config/access token=-
```

### Write to a mount path

Use the `-force` flag to write a new encryption key to the `transit` plugin:

```shell-session
$ vault write -force transit/keys/my-key
```

Write calls for `transit` requires the `-force` flag because the plugin saves
data to specific paths rather than key/value pairs.
